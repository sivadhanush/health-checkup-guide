# Annual Health Checkup Guide — Adults 24+

> A comprehensive guide to help you choose the right health checkup tests — free, open, and built for Indian adults.

**Live Site:** [Health Checkup Guide](https://sivadhanush.github.io/health-checkup-guide/)

---

## Why this exists

Every year, most of us book a "full body checkup" without really knowing:

- Which tests actually matter for our age?
- What is already covered in a standard lab package — and what is silently missing?
- What do the numbers on the report even mean?

Searching online gives you fragmented results, clinical jargon, or pages built around selling you a lab booking. There was no single, clean, honest reference written for everyday Indian adults.

So I built one — for my family first, and now for everyone.

---

## What it covers

- **25 tests** explained in plain language — from HbA1c and CBC to Vitamin D, Ferritin, B12, fasting insulin, PSA, Pap smear, and more
- **Age-wise recommendations** across five groups: 24–29, 30–39, 40–49, 50–59, and 60+
- **Three priority levels** — Mandatory, Optional, and Conditional — so you know exactly what to focus on
- **Normal reference ranges** for each test
- **Fasting requirements** clearly called out
- **Standard package coverage** — whether a test is typically included in packages from labs like Apollo, Thyrocare, and Metropolis, or needs to be added separately
- **Card view** for reading detailed explanations
- **Table view** with a custom checklist — select the tests you want and generate a compact, printable table
- **Print to PDF** — take your personalised list to your next doctor's appointment

## How to use locally

No build steps, no dependencies. Just open the file.

```bash
git clone https://github.com/your-username/health-checkup-guide.git
cd health-checkup-guide
open index.html
```

Or simply download `index.html` and open it in any browser.

---

## Tech stack

This is intentionally minimal:

- Pure HTML, CSS, and vanilla JavaScript
- Zero frameworks, zero dependencies
- Single file — everything is in `index.html`
- Hosted on GitHub Pages (free, static hosting)

The goal was to keep it as simple and long-lived as possible. No npm, no build pipeline, no maintenance overhead.
